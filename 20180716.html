语法

DISTINCT

去重操作
注意
1、distinct必须放在开头
这种写法是显然错误的
select id, distinct name from A;
但是在其他函数中distinct不受位置的影响
2、distinct作用于其后的所有字段
仅其后所有字段均相同的时候才被排除，但这种情况在实际应用中并不多。
多字段查询中对某一字段进行去重的操作
SELECT a, b, count(a) FROM A.
这种操作会产生无用列，但是这种操作代码简单，而且开销也不是是否的大。
select a,b from A group by a
利用分组来处理，在后面会详细介绍
3、distinct的效率问题
distinct在操作中一定会导致对结果排序，对性能有一定影响。所以在非必要情况下不建议使用。

WHERE

下面的运算符可在 WHERE 子句中使用：

操作符 描述
= 等于
<> 不等于
> 大于
< 小于
>= 大于等于
<= 小于等于
BETWEEN 在某个范围内
LIKE 搜索某种模式

注意
1、不等号(!=,<>)，查询出来的结果集不包含有当前字段为null的数据
2、!=不是标准的sql语句，但目前主流的数据库均可使用。

ORDER BY

ORDER BY 语句用于对结果集进行排序。
默认为升序（ASC），降序关键字为DESC
可设立多个字段进行排序，排序方式放在字段之后
